Data Management Systems

The current status of NAIC includes the management of data via custom-built tools that act as “wrappers” around backend tables in a Snowflake database. Our proposal is a flexible meta-schema which provides a complete data management pipeline, depicted in the diagram.

‹#›

Data provider invokes the Ingestion service (e.g. via API) with a submission in the specified data format (e.g. JSON, XBRL, etc.)

Ingestion service stores the form within a table in the data warehouse.

Processing service checks the a data blank, validates it against the schema corresponding to the schema that the data is aligned to.

Submission is added to the schema tables in the warehouse

Processing service runs a set of validation / transform rules that basically brins data from “raw” to “cleaned” form.

Processing service migrates clean data to a read-only storage (we propose Amazon RDS for PostgreSQL for this).

Output service provides business-ready data to the client via API. The output service directly integrates with multiple reporting tools to ready for analysis..

The information extracted from these documents may be used for pre-filling Blanks and make the submission work easier to end users.

Decision Support

The large amount of data collected by NAIC can be used for helping managers and regulators make better decisions by leveraging powerful BI tools. A typical use case of BI is to perform analysis of historical data to explain what happened, how and why. A limitation of traditional BI tools is that they provide little insight into what “may happen” in the future.

‹#›

We propose to leverage the predictive capabilities of ML-powered algorithms to fill this gap and implement BI tools that can analyze huge amounts of historical data, identify patterns in the data and make predictions on future trends. For example, historical data for “direct losses” entities incurred into, along with other related metrics, may offer insights into what went wrong.

The added value of ML-powered algorithms like those that we can build with Amazon SageMaker, is the possibility of making predictions on the likelihood of future losses, something that standard BI tools cannot help with.
